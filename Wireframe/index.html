What is the purpose of a README file?

A README file's purpose is to serve as a central, accessible text document that introduces a project, explains its contents, and provides instructions for its use, helping users, developers, and future maintainers understand and effectively utilize a software project or dataset. It's a crucial piece of documentation that includes information like the project's purpose, installation and usage instructions, contribution guidelines, licensing details, and contact information, acting as an essential guide for new users and collaborators.

What is the purpose of a wireframe?

A wireframe is a simple, visual blueprint of a digital product (website, app, or software interface).

Its key purposes are:

  1. Define Structure: Outlines the layout and placement of elements (e.g., buttons, text, images).
  2. Clarify Functionality: Shows how users interact with the interface (e.g., navigation, forms).
  3. Save Time & Costs: Quick and cheap to create, allowing early feedback and avoiding costly changes later.
  4. Improve Communication: Acts as a shared reference for designers, developers, and stakeholders.
  5. Focus on UX: Ensures the design is user-friendly before adding visual details. Guide Design: Serves as a foundation for adding colors, fonts, and visuals later.

Wireframes help teams align on functionality and layout early, ensuring a smoother design and development process.

What is a branch in Git?

A branch in Git is a lightweight, movable pointer to a specific commit in your project's history. It allows you to work on different versions of your code independently without affecting the main codebase.

  1. What is a Branch? A branch is like a separate timeline for your project. It lets you diverge from the main line of development to work on new features, bug fixes, or experiments. Each branch represents a different version of your project.
  2. Why Use Branches? Work on new features or fixes without breaking the main code. Multiple team members can work on different tasks simultaneously. Test new ideas safely. If something goes wrong, you can discard the branch without affecting the main project.
  3. How Branches Work: When you create a branch, Git creates a pointer to the current commit. As you make changes and commit them, the branch pointer moves forward. You can switch between branches to work on different tasks.
